## 1. 执行摘要
- 当前状态：健康/降级/不稳定/影响业务

- 运行角色（前端 / 核心服务 / 辅助 / 批处理）
- 依赖方向（上游 → 下游）

## 4. 当前健康状态评估

## 5. 监控信号与指标解读
表格示例：
```

```
指标类别：性能（CPU/内存/延迟/吞吐）、错误率、队列深度、连接数、磁盘、GC、线程、会话、请求分布等。

## 6. 日志 / 事件 / 错误模式
- 错误类型分类（超时 / 5xx / 限流 / 认证失败 / 连接重置）
- 异常堆栈或签名（脱敏 / 抽象化）
- 最近变更 / 发布 / 配置调整
- 平台事件（维护 / 区域性波动 / 依赖故障）

## 7. 性能与容量分析
- 瓶颈类型（CPU 受限 / IO 受限 / 内存碎片 / 网络拥塞）

## 8. 配置与架构风险
- 尺寸 / SKU 不匹配

## 9. 外部依赖与系统交互影响
- 下游依赖延迟或错误传播

- 第三方服务不稳定传播链

## 可用 KQL 模板（按需引用）
```
// 最近错误类型分布
AppTraces
| where TimeGenerated > ago(2h)
| where SeverityLevel >= 3
| summarize count() by Message
| top 20 by count_
```

```
// 请求延迟分位数
requests
| where TimeGenerated > ago(1h)
| summarize p50(duration), p95(duration), p99(duration) by bin(TimeGenerated, 5m)
```

```
// 依赖调用失败率
dependencies
| where TimeGenerated > ago(1h)
| summarize failures=sumif(itemCount, success == false), total=sum(itemCount) by target
| extend failureRate = failures * 100.0 / total
| top 10 by failureRate desc
```
