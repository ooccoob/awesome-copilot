## 最佳实践

- **Records**：主要用于存储数据的类（如 DTO、不可变数据结构）应优先用 Java Record。
- **模式匹配**：用 instanceof 和 switch 表达式的模式匹配简化条件逻辑和类型转换。
- **类型推断**：局部变量声明用 var 提升可读性，但仅在类型明确时使用。
- **不可变性**：优先使用不可变对象。类和字段尽量 final。固定数据用 List.of()/Map.of()，用 Stream.toList() 创建不可变列表。
- **Streams 和 Lambda**：集合处理用 Streams API 和 lambda 表达式，优先用方法引用（如 stream.map(Foo::toBar)）。
- **空值处理**：避免返回或接收 null。可用 Optional<T> 表示可空值，善用 Objects 工具类（如 equals、requireNonNull）。

### 命名规范
- 遵循 Google Java 风格：
  - 类/接口名用 UpperCamelCase
  - 方法/变量名用 lowerCamelCase
  - 常量用 UPPER_SNAKE_CASE
  - 包名用小写
- 类名用名词（如 UserService），方法用动词（如 getUserById）
- 避免缩写和匈牙利命名
