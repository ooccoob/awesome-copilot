# 执行计划

## 概述

加载计划，批判性审查，按批次执行任务，在批次间报告审查。

**核心原则：** 带检查点的批次执行供架构师审查。

**开始时宣布：** "我正在使用执行计划技能来实现这个计划。"

## 过程

### 步骤1：加载和审查计划

1. 阅读计划文件
2. 批判性审查 - 识别计划的任何问题或关切
3. 如果有关切：在开始前向您的人类合作伙伴提出
4. 如果没有关切：创建TodoWrite并继续

### 步骤2：执行批次

**默认：前3个任务**

对于每个任务：

1. 标记为in_progress
2. 严格按照每个步骤执行（计划有小步骤）
3. 按指定运行验证
4. 标记为完成

### 步骤3：报告

批次完成时：

- 显示实现了什么
- 显示验证输出
- 说："准备好接收反馈。"

### 步骤4：继续

基于反馈：

- 如需要应用更改
- 执行下一批次
- 重复直到完成

### 步骤5：完成开发

所有任务完成并验证后：

- 宣布："我正在使用完成开发分支技能来完成这项工作。"
- **必需子技能：** 使用superpowers:finishing-a-development-branch
- 遵循该技能验证测试，展示选项，执行选择

## 何时停止并请求帮助

**立即停止执行当：**

- 在批次中遇到阻塞器（缺少依赖、测试失败、指令不清楚）
- 计划有严重缺口阻止开始
- 您不理解指令
- 验证重复失败

**请求澄清而不是猜测。**

## 何时重新访问早期步骤

**返回审查（步骤1）当：**

- 合作伙伴基于您的反馈更新计划
- 根本方法需要重新思考

**不要强行通过阻塞器** - 停止并询问。

## 记住

- 首先批判性审查计划
- 严格按照计划步骤执行
- 不要跳过验证
- 当计划说时引用技能
- 批次间：仅报告并等待
- 被阻塞时停止，不要猜测
