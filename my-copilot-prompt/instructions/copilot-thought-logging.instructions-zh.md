# Copilot 进程跟踪说明

**绝对强制性规则：**
- 在执行任何步骤之前，您必须完整阅读这些说明，以了解完整的说明指南。
- 您必须严格按照规定遵循这些说明，不得有任何偏差。
- 除非明确要求，否则在处理或解释时不要不断重复状态更新。这很糟糕，会淹没 Copilot 会话上下文。
- 无阶段公告（输出中无“# Phase X”标头）
- 必须按照指定的确切顺序一次执行一个阶段。
- 一个响应中没有阶段组合
- 无需跳过阶段
- 没有详细的解释或评论
- 只输出阶段指令中指定的确切文本

# 第一阶段：初始化

- 在工作区根目录中创建文件 `\Copilot-Processing.md`
- 使用用户请求详细信息填充 `\Copilot-Processing.md`
- 默默地工作，没有公告，直到完成。
- 当此阶段完成时，请记住<阶段 1> 已完成并且不需要重复。

# 第二阶段：规划

- 将行动计划生成到 `\Copilot-Processing.md` 文件中。
- 生成详细且细粒度的任务特定操作项，用于跟踪文件 `\Copilot-Processing.md` 中的每个操作计划项以及待办事项/完成状态。
- 这应该包括：
  - 行动计划中每个行动项目的具体任务作为一个阶段。
  - 清楚地描述需要做什么
  - 每项任务的任何依赖项或先决条件
  - 确保任务足够细粒度，可以一次执行一项
- 默默地工作，没有公告，直到完成。
- 当此阶段完成时，请记住<阶段 2> 已完成并且不需要重复。

# 第三阶段：执行

- 按逻辑分组/阶段执行行动计划中的行动项目
- 默默地工作，没有公告，直到完成。
- 更新文件 `\Copilot-Processing.md` 并在跟踪中将操作项标记为完成。
- 当一个阶段完成时，请记住 `\Copilot-Processing.md` 中的特定阶段已完成并且不需要重复。
- 重复此模式，直到完成所有操作项

# 第四阶段：总结

- 将摘要添加到 `\Copilot-Processing.md`
- 默默地工作，没有公告，直到完成。
- 仅当所有操作完成时执行
- 通知用户：“已将最终摘要添加到 `\Copilot-Processing.md`。”
- 提醒用户查看摘要并确认该过程完成，然后在完成后删除该文件，以免将其添加到存储库中。

**执行规则：**
- 切勿在响应中写入“# Phase X”标头
- 除非明确要求，否则切勿在输出中重复单词“Phase”
- 切勿提供超出指定确切文本的解释
- 切勿在一个响应中合并多个阶段
- 在没有用户输入的情况下，切勿继续过去的当前阶段
- 如果您发现自己过于冗长，请停止并仅提供所需的输出
- 如果您发现自己要跳过某个阶段，请停止并返回到正确的阶段
- 如果您发现自己正在组合阶段，请停止并仅执行当前阶段
