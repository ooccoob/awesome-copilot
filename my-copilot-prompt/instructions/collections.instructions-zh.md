# 馆藏开发

## 领取须知

使用 Awesome-copilot 存储库中的集合时：

- 在提交之前始终使用 `node validate-collections.js` 验证集合
- 遵循为集合清单建立的 YAML 架构
- 仅引用存储库中的现有文件
- 使用包含小写字母、数字和连字符的描述性集合 ID
- 让集合集中于特定的工作流程或主题
- 测试所有引用的项目是否可以很好地协同工作

## 集合结构

- **必填字段**：id、名称、描述、项目
- **可选字段**：标签、显示
- **项目要求**：路径必须存在，种类必须与文件扩展名匹配
- **显示选项**：排序（alpha/手动），show_badge（真/假）

## 验证规则

- 集合 ID 在所有集合中必须是唯一的
- 文件路径必须存在且与项目类型匹配
- 标签只能使用小写字母、数字和连字符
- 集合必须包含 1-50 项
- 描述必须为 1-500 个字符

## 最佳实践

- 组 3-10 相关项目以获得最佳可用性  
- 使用清晰的描述性名称和描述
- 添加相关标签以提高可发现性
- 测试集合支持的完整工作流程
- 确保项目有效互补

## 文件组织

- 集合不需要重新组织文件
- 项目可以位于存储库中的任何位置
- 使用存储库根目录的相对路径
- 维护现有的目录结构（prompts/、instructions/、agents/）

## 生成过程

- 集合通过 `npm start` 自动生成 README 文件
- 单独的集合页面在 collections/ 目录中创建
- 主要集合概述生成为 README.collections.md
- 为每个项目自动创建 VS Code 安装徽章
