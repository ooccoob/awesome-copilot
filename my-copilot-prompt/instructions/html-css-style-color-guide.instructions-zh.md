# HTML CSS 样式颜色指南

更新或创建用于浏览器呈现的 HTML/CSS 样式时请遵循这些准则。颜色名称
代表其各自色调范围的全光谱（例如，“蓝色”包括海军蓝、天蓝色等）。

## 颜色定义

- **热门颜色**：橙色、红色和黄色
- **冷色**：蓝色、绿色和紫色
- **中性色**：灰色和灰度变化
- **二元色**：黑色和白色
- **60-30-10 规则**
  - **原色**：使用 60% 的时间（*冷色或浅色*）
  - **次要颜色**：使用 30% 的时间（*冷色或浅色*）
  - **强调色**：使用 10% 的时间（*补充热门颜色*）

## 颜色使用指南

通过将**60-30-10规则**应用于背景等图形设计元素来平衡所使用的颜色，
按钮、卡片等...

### 背景颜色

**切勿使用：**

- 紫色或洋红色
- 红色、橙色或黄色
- 粉色
- 任何大热色

**推荐：**

- 白色或灰白色
- 浅冷色（例如浅蓝色、浅绿色）
- 微妙的中性色调
- 具有最小色移的光渐变

### 文字颜色

**切勿使用：**

- 黄色（对比度和可读性较差）
- 粉色
- 浅色背景上的纯白色或浅色文本
- 深色背景上的纯黑色或深色文本

**推荐：**

- 深色中性色（例如#1f2328、#24292f）
- 近黑色变体（#000000 至 #333333）
  - 确保背景为浅色
- 深灰色（#4d4d4d、#6c757d）
- 高对比度组合，方便使用
- 近白色变体（#ffffff 到 #f0f2f3）
  - 确保背景是深色

### 应避免的颜色

除非设计规范或用户要求明确要求，否则应避免：

- 明亮的紫色和洋红色
- 明亮的粉色和霓虹色
- 高饱和度的热色
- 对比度低的颜色（不符合 WCAG 可访问性标准）

### 谨慎使用的颜色

**热门颜色**（红色、橙色、黄色）：

- 保留重要警报、警告或错误消息
- 仅在表达紧急性或重要性时使用
- 仅限于小的重点区域而不是大的部分
- 在使用热色之前考虑图标或粗体文本等替代方案

## 渐变

应用具有微妙颜色过渡的渐变以保持专业美感。

### 最佳实践

- 保持颜色变化最小（例如，#E6F2FF 到 #F5F7FA）
- 使用同一颜色系列内的渐变
- 避免在单一渐变中组合冷色和热色
- 对于背景，更喜欢线性渐变而不是径向渐变

### 适当的用例

- 背景容器和部分
- 按钮悬停状态和交互元素
- 阴影和深度效果
- 标题和导航栏
- 卡组件和面板
