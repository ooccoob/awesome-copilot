# minipro 芯片编程实用程序规范

## 1. 概述

**minipro** 是一款开源命令行实用程序，用于使用受支持的通用编程器（例如 **T48**、**TL866II Plus** 和兼容型号）**对各种 **EEPROM、闪存、EPROM、SRAM、GAL 和逻辑器件**进行编程、读取、擦除和验证。

它广泛用于 **Linux**、**macOS** 和 **Windows** 环境，特别是用于 **逆计算**、**固件开发** 和 **电子原型制作**。

---

## 2. 支持的程序员

|程序员|笔记|
| ------------ | -------------------------- |
| T48|全力支持（推荐）|
| TL866II加|全力支持|
| TL866A/CS |有限/遗留支持|

---

## 3. 支持的设备类型

### 3.1 存储设备

* 并行 EEPROM（例如 AT28C256）
* 闪存（29xxx系列）
* EPROM（27xxx 系列）
* SRAM（仅读取/验证）

### 3.2 逻辑和PLD

* GAL16V8 / GAL22V10
* PAL 设备（有限）

### 3.3 其他设备

* 一些微控制器（取决于设备）
* 逻辑IC测试（选定型号）

## 4. 安装

### 4.1 Linux

```bash
sudo apt install minipro
```

或来自来源：

```bash
git clone https://github.com/vdudouyt/minipro.git
make
sudo make install
```

### 4.2 窗口

* 通过 MSYS2 或预构建的二进制文件安装
* 需要 libusb 驱动程序 (WinUSB)

## 5. 基本命令语法

```bash
minipro [options]
```

常用选项：

|选项 |描述 |
| ------------- | ---------------------- |
| __代码0__ |选择目标设备 |
| __代码0__ |读取设备到文件 |
| __代码0__ |将文件写入设备|
| __代码0__ |擦除设备|
| __代码0__ |验证内容 |
| __代码0__ |设备信息|
| __代码0__ |列出支持的设备 |

---

## 6. 常用编程操作

### 6.1 列出支持的设备

```bash
minipro -l
```

### 6.2 识别设备

```bash
minipro -p AT28C256 -I
```

### 6.3 读取芯片

```bash
minipro -p AT28C256 -r rom_dump.bin
```

### 6.4 烧写芯片

```bash
minipro -p AT28C256 -w rom.bin
```

### 6.5 仅验证

```bash
minipro -p AT28C256 -v rom.bin
```

---

## 7. EEPROM 编程（AT28C256 示例）

```bash
minipro -p AT28C256 -w monitor.bin
```

* 自动处理软件数据保护
* 写周期延迟由内部管理
* 编程后执行验证

---

## 8. 闪存编程

```bash
minipro -p SST39SF040 -e -w firmware.bin
```

* 闪存设备所需的擦除步骤
* 自动处理扇区擦除

---

## 9. EPROM 操作

```bash
minipro -p 27C256 -r eprom.bin
```

* 重新编程前需要进行 UV 擦除
* minipro 在写入前验证空白状态

---

## 10.GAL编程

```bash
minipro -p GAL22V10 -w logic.jed
```

* 使用 JEDEC 文件
* 支持读、写、验证
* 可通过 `-I` 查看保险丝图

| ---------------------- | -------------------------- |
| __代码0__ |设备选择错误 |
| __代码0__ |数据不匹配|
| __代码0__ |写保护已启用|
| __代码0__ |插入或接线错误 |

---

## 12. 安全和最佳实践

* 始终确认 ZIF 插座中的设备方向
* 使用正确的设备标识符 (`-p`)
* 运行时请勿热插芯片
* 使用适用于 PLCC、SOP、TSOP 封装的适配器

---

## 13. 典型的逆计算工作流程

1. 组装 ROM 映像
2. 使用 minipro + T48 对 EEPROM 进行编程
3. 验证内容
4. 在SBC中安装芯片
5. 测试系统启动

---

## 14. 限制

* 并非所有设备都受支持
* 一些微控制器需要专有工具
* 不支持在线编程 (ISP)
