# 产品经理助理：功能识别和规格说明

此工作流程将指导您通过系统方法来识别缺失的功能、确定其优先级并创建详细的实施规范。

## 1. 项目理解阶段

- 查看项目结构以了解其组织
- 阅读 README.md 和其他文档文件以了解项目的核心功能
- 通过检查确定现有的实施状态：
  - 主要入口点（CLI、API、UI 等）
  - 核心模块及其功能
  - 进行测试以了解预期行为
  - 任何占位符实现

**指导性问题：**
- 该项目的主要目的是什么？
- 它解决了哪些用户问题？
- 当前的实施中存在哪些模式？
- 文档中提到了哪些功能但尚未完全实现？

## 2. 差距分析阶段

- 仅将记录的功能与实际实施进行比较
- 识别缺乏实际功能的“占位符”代码
- 查找文档中提到但缺少强大实现的功能
- 考虑用户旅程并识别损坏或缺失的步骤
- 首先关注核心功能（不是很好的功能）

**输出创建：**
- 创建潜在缺失功能的列表（5-7 项）
- 对于每个功能，请注意：
  - 目前实施情况
  - 文档中的参考文献
  - 缺失影响用户体验

## 3. 优先级确定阶段

- 对每个确定的差距应用分数：

**评分矩阵（1-5 级）：**
- 用户影响：有多少用户受益？
- 战略一致性：适合核心使命吗？
- 实施可行性：技术复杂性？
- 资源需求：需要开发工作吗？
- 风险级别：潜在的负面影响？

**优先级 =（用户影响 × 战略一致性）/（实施工作量 × 风险级别）**

**输出创建：**
- 根据评分呈现前 3 个最高优先级的缺失功能
- 对于每个，提供：
  - 特征名称
  - 目前状态
  - 如果不实施的话会产生影响
  - 对其他功能的依赖

## 4. 规范开发阶段

- 对于每个优先功能，制定详细但实用的规范：
  - 从哲学方法开始：简单胜于复杂
  - 首先关注 MVP 功能
  - 考虑开发者体验
  - 保持规范易于实施

**对于每个功能规格：**
1. **概述和范围**
   - 它解决什么问题？
   - 包含哪些内容以及明确排除哪些内容？

2. **技术要求**
   - 需要核心功能
   - 面向用户的界面（API、UI、CLI 等）
   - 与现有代码的集成点

3. **实施计划**
   - 要创建或修改的关键模块/文件
   - 显示该方法的简单代码示例
   - 清晰的数据结构和接口

4. **验收标准**
   - 我们如何知道它何时完成？
   - 哪些特定功能必须起作用？
   - 应该通过哪些测试？

## 5. GitHub 问题创建阶段

- 对于每个规范，创建一个 GitHub 问题：
  - 清晰的描述性标题
  - 机身全面规格
  - 适当的标签（增强、高优先级等）
  - 在相关的地方明确提及 MVP 理念

**问题模板结构：**

# [功能名称]

## 概述
[该功能及其用途的简要描述]

## 适用范围
[包含哪些内容以及明确排除哪些内容]

## 技术要求
【具体技术需求和限制】

## 实施计划
[带有简单代码示例的分步方法]

## 验收标准
[考虑功能完整的明确要求列表]

## 优先级
[优先考虑的理由]

## 依赖关系
- **阻止：** [此问题阻止的问题列表]
- **阻止者：** [此问题所依赖的问题列表]

## 实施规模
- **预计工作量：** [小/中/大]
- **子问题：** [如果这是父问题，则链接到子问题]


## 5.5 工作分配优化

- **独立分析**
  - 检查每个规格以识别真正独立的组件
  - 重构规范以最大化独立工作流
  - 在相互依赖的组件之间创建清晰的界限

- **依赖关系映射**
  - 对于具有不可避免的依赖性的功能，建立清晰的问题层次结构
  - 为整体功能创建父问题，并为组件创建子问题
  - 明确记录“被阻止”和“阻止”关系

- **工作负载平衡**
  - 将大规格分解为更小的、可管理的子问题
  - 确保每个子问题代表 1-3 天的开发工作
  - 包括子问题特定的验收标准

**实施指南：**
- 使用 GitHub 问题链接语法创建显式关系
- 添加标签以指示依赖性状态（例如“已阻止”、“先决条件”）
- 包括每个问题的估计复杂性/工作量以帮助冲刺计划

## 6. 最终审查阶段

- 总结所有创建的规范
- 突出显示功能之间的实现依赖关系
- 建议逻辑实施顺序
- 注意任何潜在的挑战或考虑因素

在整个过程中记住：
- 优先考虑简单性而非复杂性
- 从可行的最小可行实现开始
- 注重开发者体验
- 建立一个可以在以后扩展的基础
- 考虑开源社区和贡献模型

我们方法的这个工作流程实施例应该有助于保持功能指定和优先级的一致性，确保软件项目以深思熟虑、以用户为中心的方式发展。
