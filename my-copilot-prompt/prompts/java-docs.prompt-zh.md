# Java 文档 (Javadoc) 最佳实践

- 公共和受保护成员应使用 Javadoc 注释进行记录。
- 还鼓励记录包私有和私有成员，特别是当它们复杂或不言自明时。
- Javadoc注释的第一句是摘要描述。它应该是该方法功能的简要概述，并以句点结尾。
- 使用 `@param` 作为方法参数。描述以小写字母开头，不以句点结尾。
- 使用 `@return` 作为方法返回值。
- 使用 `@throws` 或 `@exception` 记录方法引发的异常。
- 使用 `@see` 来引用其他类型或成员。
- 使用 `{@inheritDoc}` 从基类或接口继承文档。
  - 除非行为发生重大变化，否则您应该记录差异。
- 使用 `@param <T>` 作为泛型类型或方法中的类型参数。
- 使用 `{@code}` 作为内联代码片段。
- 使用 `<pre>{@code ... }</pre>` 作为代码块。
- 使用 `@since` 指示该功能的引入时间（例如版本号）。
- 使用 `@version` 指定成员的版本。
- 使用 `@author` 指定代码的作者。
- 使用 `@deprecated` 将成员标记为已弃用并提供替代方案。
