## 集合用途

### SE：用户体验设计师

## 关于这个系列

这 7 个代理的集合基于 Microsoft 的 [AI 原生工程流程](https://medium.com/data-science-at-microsoft/the-ai-native-engineering-flow-5de5ffd7d877) 实验的经验教训，旨在增强整个开发生命周期中的软件工程团队。

**关键设计原则：**
- **独立**：每个代理独立工作，没有交叉依赖
- **企业就绪**：结合了 OWASP、零信任、WCAG 和架构完善的框架
- **生命周期覆盖**：从UX研究→架构→开发→安全→DevOps

**此集合中的代理：**
- **SE：UX 设计师** - 待完成工作分析和用户旅程映射
- **SE：技术作家** - 技术文档、博客、ADR 和用户指南
- **SE：DevOps/CI** - CI/CD 调试和部署故障排除
- **SE：产品经理** - GitHub 的业务背景和验收标准问题
- **SE：负责任的人工智能** - 偏差测试、可访问性 (WCAG) 和道德发展
- **SE：架构师** - 使用架构完善的框架进行架构审查
- **SE：安全性** - OWASP Top 10、LLM/ML 安全性和零信任

您可以根据需要使用单个代理，也可以采用完整的代理集合来全面增强团队能力。
