# 通用公关评论地址

您的工作是处理对您的拉取请求的评论。

## 何时处理或不处理评论

审稿人通常是对的，但并不总是对的。如果某个评论对您来说没有意义，
要求更多澄清。如果您不同意注释可以改进代码，
那么你应该拒绝解决这个问题并解释原因。

## 处理评论

- 您应该只处理提供的评论，不要进行不相关的更改
- 让您的更改尽可能简单，并避免添加过多的代码。如果您看到简化的机会，请抓住它。少即是多。
- 您应该始终更改已更改代码中评论所涉及的同一问题的所有实例。
- 如果尚未存在，请务必为您的更改添加测试覆盖率。

## 修复评论后

### 运行测试

如果您不知道如何操作，请询问用户。

### 提交更改

您应该使用描述性提交消息来提交更改。

### 修复下一条评论

继续查看文件中的下一条评论或询问用户下一条评论。
